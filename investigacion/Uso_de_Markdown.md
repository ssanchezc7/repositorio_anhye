# Uso de Markdown

## ¿Qué es Markdown?
Markdown es un lenguaje de marcado ligero que permite formatear texto plano usando una sintaxis sencilla. Se utiliza ampliamente en documentación técnica, README y wikis.

## ¿Por qué usar Markdown en proyectos de software?
- Es fácil de leer en bruto y procesable por herramientas (GitHub, GitLab).
- Permite versionado eficiente junto con el código en repositorios.
- Fácil conversión a HTML, PDF o DOCX con herramientas como Pandoc.

## Ejemplo práctico

### Encabezados
# Encabezado 1
## Encabezado 2

### Listas
- Elemento 1
- Elemento 2
  - Sub-elemento

### Tabla
| Campo | Descripción |
|---|---|
| ISBN | Identificador del libro |
| Título | Título del libro |

### Enlace e imagen
- Enlace: [GitHub](https://github.com)
- Imagen: `![alt text](ruta/a/imagen.png)` (usa ruta relativa dentro del repo)

## Ventajas de usar Markdown con GitHub
- Renderizado automático de archivos `.md` en la interfaz.
- Facilita la revisión por commits y pull requests.
- Permite mantener documentación cerca del código y en sincronía con versiones.
