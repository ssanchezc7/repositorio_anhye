# Sistema de Gestión de Biblioteca

**Descripción:**
- Sistema para gestionar el catálogo de libros, préstamos, devoluciones, usuarios y reportes en una biblioteca universitaria.

**Objetivos:**
- Facilitar el registro y búsqueda de libros.
- Gestionar préstamos y devoluciones con control de vencimientos.
- Mantener registro de usuarios y su historial de préstamos.
- Generar reportes básicos (libros más prestados, morosidad, stock).

**Requerimientos principales:**
- Ver `requerimientos/DRS_v1.md` y `requerimientos/DRS_v2.md` para versiones detalladas.

**Plan de pruebas (resumen):**
- Ver `pruebas/PlanPruebas.md` con casos de prueba, resultados esperados y validación.

**Mantenimiento propuesto:**
- Tipo: Perfectivo (mejoras continuas) y Correctivo (corrección de errores), ver `mantenimiento/Propuesta_Mantenimiento.md`.

**Investigación:**
- Uso de Markdown y ventajas en `investigacion/Uso_de_Markdown.md`.

**Evidencias:**
- `evidencias/historial_commits.png` (historial de commits)
- `evidencias/versionado_docs.png` (ejemplo de versiones de documentos)
- `evidencias/estructura_repo.png` (estructura del repositorio)
